Importing Data

Antes de poder ejecutar anlisis de datos o crear grficos en R, debemos importar esos datos a R. Preparar y limpiar los datos para el anlisis es esencial y a menudo requiere ms tiempo que los propios anlisis estadsticos. Es inusual que los datos sin procesar estn en el formato correcto y sin errores. La limpieza de datos es el proceso de identificar y corregir cualquier problema para que los datos puedan ser analizados fcilmente.

Importar datos es un desafo importante para los principiantes. Este mdulo brindar instrucciones para una de las formas ms comunes de importar datos, as como algunos desafos que puedes enfrentar y cmo superarlos. Algunos de estos problemas se pueden evitar siguiendo buenas prcticas de entrada y gestin de datos (lee primero Entrada de datos para obtener ayuda con esto).

Importar datos como un marco de datos

Recomendamos ingresar los datos en un programa de hoja de clculo (por ejemplo, Excel) y guardar esos datos como un archivo de valores separados por comas (.csv). Estos se pueden leer fcilmente en R y compartir entre usuarios con diferentes programas de hojas de clculo.

En este mdulo, vamos a utilizar un conjunto de datos de muestra sobre el comportamiento de alimentacin de un caracol marino para demostrar cmo importar los datos y los problemas ms comunes que surgen al importar y limpiar datos en R.

Primero, guarda el archivo de datos, Snail_feeding.csv, en tu directorio de trabajo. Consulta Introduccin a R para obtener ayuda sobre cmo configurar el directorio de trabajo.

Segundo, importa el archivo de datos a un marco de datos llamado Snail_feeding utilizando la funcin read.csv.

Snail_feeding <- read.csv("Snail_feeding.csv")

Limpieza de data frames

Cuando utilizas read.csv, R utiliza varios argumentos por defecto que se pueden modificar para asegurar que tus datos se importen con menos errores. Echa un vistazo al archivo de ayuda dentro de R (escribiendo ?read.csv) para familiarizarte con algunos de estos argumentos.

Los que son particularmente tiles son:

header = T - al especificar esto como T (es decir, TRUE), asegurars que los valores en la primera fila de tus datos se traten como nombres de variables.

strip.white = T - esto eliminar los espacios en blanco al final y al principio de los factores. Este es un error comn al escribir los datos (por ejemplo, males vs males_). Si lo establecemos como TRUE, ambos se convertirn en males. De lo contrario, R considerar que hay dos niveles diferentes.

na.strings = "" - esto asegurar que las celdas vacas se reemplacen por NA (la forma en que R registra los datos faltantes). De manera molesta, R importa valores faltantes dentro de caracteres/factores como el valor . Usando na.strings = "", se insertarn NAs en su lugar. Adems, si has codificado valores faltantes como algo distinto a un espacio en blanco, puedes definir ese valor faltante usando este argumento (por ejemplo, na.strings = c("", "-", "*")).

Al combinar todos estos en la funcin read.csv, obtendremos un marco de datos ms limpio.

Snail_feeding <- read.csv("Snail_feeding.csv", header = T, strip.white = T, na.strings = "")

Problemas comunes al importar datos

No obtienes el nmero de columnas o filas que esperas

Si ves un montn de columnas adicionales (X, X.1, X.2, X.3, etc.) o filas en tu marco de datos llenas de NA, es probable que se haya ingresado un carcter adicional (probablemente un espacio en blanco o una tabulacin) en celdas ms all de tus datos reales en Excel. Este problema se puede evitar durante la entrada de datos eliminando todos los colores/formatos y vaciando columnas/filas excepto las necesarias.

Si el problema persiste, podemos abordarlo en R mediante la indexacin dentro de corchetes cuadrados dataframe[fila, columna]. Por ejemplo, el siguiente cdigo reemplazar el marco de datos en este ejemplo con uno que solo incluye las primeras 7 columnas y elimina las 5 no deseadas (X, X.1, X.2, X.3, etc.).

Snail_feeding <- Snail_feeding[, 1:7]

El paquete dplyr tiene varias funciones muy tiles para subconjuntar filas y columnas. Consulta Subsetting data para obtener ayuda con esto.

Si tienes muchas columnas y no quieres contarlas, puedes usar lgica (&, or, ==, >, < , !=) para eliminar las filas y columnas no deseadas. No queremos eliminar todos los valores NA, solo las filas y columnas adicionales que estn completamente llenas de NAs. Estas tienen la propiedad de que el nmero de NAs es igual al nmero de filas en una columna dada (o nmero de columnas en una fila dada).

Hacemos esto seleccionando las columnas de nuestro data frame donde la suma de columnas, colSums, de todos los NAs, >is.na, en nuestro data frame no sea igual, !=, al nmero de filas, nrow, de nuestro data frame. Puedes usar la misma lgica para hacer esto con las filas tambin.

# Selects rows that are not entirely NA
Snail_feeding <- Snail_feeding[, colSums(is.na(Snail_feeding)) != nrow(Snail_feeding)]
# Select columns that are not entirely NA
Snail_feeding <- Snail_feeding[rowSums(is.na(Snail_feeding)) != ncol(Snail_feeding), ]

Las columnas no son del tipo de datos que esperas

Caracteres como factores Cuando se importan marcos de datos en R, los caracteres se convierten automticamente en factores. Esto tiene mucho sentido para el trabajo estadstico; es ms probable que queramos trabajar con factores (definidos anteriormente) que con caracteres. Los factores son fantsticos cuando se hace anlisis estadstico y se explora realmente los datos. Sin embargo, antes de eso, cuando se leen, limpian, solucionan problemas, fusionan y manipulan los datos en general, los factores son un dolor de cabeza total.

Puedes usar el argumento stringsAsFactors=FALSE al importar los datos para evitar la conversin automtica a factores o puedes usar as.character para convertir vectores individuales en caracteres.

Consejo: Utiliza factores lo ms tarde posible en el proceso de anlisis. Configura stringsAsFactors=FALSE al importar tus datos y, cuando necesites factores en algn lugar dentro de tu script, convierte los datos a un factor.

Factores como enteros En este marco de datos, el ID de caracol es un entero cuando debera ser un factor o un carcter. Esto se debe a que en la hoja de datos original, el ID de caracol se codific utilizando nmeros. Este es un problema comn y es fcil de solucionar. Simplemente usa as.factor() o as.character(), y luego class() para verificar que nuestra conversin haya funcionado. Recuerda usar $ para acceder al vector desde dentro del marco de datos.

Snail_feeding$Snail <- as.factor(Snail_feeding$Snail)
class(Snail_feeding$Snail)
## [1] "factor"

Nmeros como caracteres (factores) Debido a la coercin automtica en R, cualquier dgito no numrico dentro de una variable numrica resultar en que toda la variable se convierta en un carcter (R lo importa como factores). En este ejemplo, la variable Distance se ha importado como un factor con 768 niveles, cuando en realidad debera ser una variable numrica. Esto indica que hay un error tipogrfico en alguna parte del vector de distancias. Estos errores tipogrficos son comunes (por ejemplo, usar accidentalmente una coma en lugar de un punto decimal al ingresar decimales), pero un poco ms difciles de resolver.

Con un conjunto de datos pequeo, lo ms rpido es regresar a los datos originales y encontrar el error. Una vez corregido, vuelves a importar y la variable debera ser numrica.

Con un conjunto de datos ms grande, surge el problema de tener que encontrar los errores tipogrficos. Desafortunadamente, no podemos convertir directamente nuestros datos de un vector de tipo factor a numrico. Esto se debe al atributo levels (consultar Tipos de datos y estructura). Si intentas convertir un factor directamente en una variable numrica, los valores se convierten en un nmero que corresponde al nmero de niveles (1:n, ordenados alfabticamente o de forma ascendente) en lugar del valor real.

Por lo tanto, primero debemos convertirlo a carcter y luego a numrico. Al convertirlo a numrico, obtendrs un mensaje de advertencia NAs introduced by coercion. Esto se debe a que los valores no numricos (es decir, nuestros errores tipogrficos) no se pueden convertir a numricos y se reemplazan por NA.

Podemos aprovechar esto, utilizando is.na en combinacin con which para identificar dnde estn los errores tipogrficos o los valores faltantes dentro del vector.

Snail_feeding$Distance <- as.character(Snail_feeding$Distance)
Snail_feeding$Distance <- as.numeric(Snail_feeding$Distance)
## Warning: NAs introducidos por coercin

which(is.na(Snail_feeding$Distance))
## [1] 682 755

Esto nos indica que algo extrao sucedi en las filas 682 y 755. Ahora que hemos identificado dnde est el problema, es fcil reemplazar los valores en nuestro marco de datos con los valores correctos (vuelve a tu hoja de datos original). Puedes corregir el error en el conjunto de datos y volver a importarlo, o reemplazar los valores en R utilizando ndices dentro de corchetes y asignando el nuevo valor con <-. Usa which(is.na()) para verificar si tu correccin funcion.

Snail_feeding[682, "Distance"] <- 0.356452
Snail_feeding[755, "Distance"] <- 0.42336

which(is.na(Snail_feeding$Distance))
## integer(0)

Tienes ms niveles de variables de los esperados

Uno de los pasos ms importantes en cualquier anlisis de datos o tarea de procesamiento es verificar que los valores de tus datos sean correctos. Por ejemplo, se esperara que una variable llamada Sexo tenga solo dos niveles. Sin embargo, en nuestro marco de datos tiene cinco niveles (ver str y summary arriba).

Puedes verificar los niveles de un factor o los valores nicos de caracteres con levels (solo para factores) o unique (para caracteres y factores).

levels(Snail_feeding$Sex)
## NULL
unique(Snail_feeding$Sex)
## [1] "male"     "males"    "Male"     "female"   "female s"

Hay varios errores tipogrficos que podemos corregir utilizando unique y which, as como los operadores lgicos == (igual) y | (o) para identificar y reemplazar los errores tipogrficos.

Para reemplazar cualquier valor que sea males o Male con male, usaramos:

Snail_feeding$Sex[which(Snail_feeding$Sex == "males" | Snail_feeding$Sex == "Male")] <- "male"

Para reemplazar cualquier valor que sea female s con female, usaramos:

Snail_feeding$Sex[which(Snail_feeding$Sex == "female s")] <- "female"

Verifica que funcione usando unique, pero tambin observa lo que sucede cuando verificas los levels (niveles).

unique(Snail_feeding$Sex)
## [1] "male"   "female"
levels(Snail_feeding$Sex)
## NULL

Cuando usamos unique para verificar nuestras categoras, solo aparecen male y female, sin embargo, cuando observamos los niveles, todava tenemos los cinco niveles diferentes, incluyendo los errores tipogrficos. Esto se debe al comportamiento de los factores. Una vez que se han definido los niveles, seguirn existiendo independientemente de si estn incluidos en alguna muestra. Dado que nuestros niveles adicionales eran errores tipogrficos y no niveles reales, debemos eliminarlos de los atributos.

factor eliminar los niveles adicionales de un vector.

Snail_feeding$Sex <- factor(Snail_feeding$Sex)
levels(Snail_feeding$Sex)
## [1] "female" "male"

Errores numricos

Usar summary anteriormente es una herramienta til para verificar posibles errores tipogrficos en nuestras variables numricas. Compara el valor mximo o mnimo con la mediana de cada variable numrica. Si alguno de los valores es una orden de magnitud mayor o menor que la mediana, podra ser un error tipogrfico. Por ejemplo, observa la profundidad mxima, parece que se ha olvidado el punto decimal.

Nuevamente, podemos utilizar operadores lgicos e indexacin para identificar posibles errores numricos. Dado que todos los valores de nuestra variable de profundidad parecen estar entre 1 y 2 (segn los rangos intercuartiles de summary), buscaremos filas con una profundidad mayor a 2.

Snail_feeding[which(Snail_feeding$Depth > 2), ]
##   Snail.ID  Sex  Size Feeding Distance Depth Temp Snail
## 8        1 male small    TRUE      0.6   162   20     1

Solo hay 1 fila. Despus de confirmar con nuestros datos originales que esto es, de hecho, un error tipogrfico, lo reemplazaremos con el valor real. Seleccionando la fila y la columna, reemplazamos el valor 162 con 1.62.

Snail_feeding[which(Snail_feeding$Depth > 2), 6] <- 1.62

Por qu hacer esto en R?

Puede que te preguntes por qu molestarse en corregir esto en R. Por qu no ir directamente al archivo .csv y corregir todos los problemas? Por lo general, est bien hacer esto si sabes que el problema es un error tipogrfico, etc. Sin embargo, para mantener la integridad de los datos, es importante tener un registro de cada cambio que se haya realizado en tu conjunto de datos original. Al realizar todas las manipulaciones y correcciones en R, tambin ests guardando un registro de todos los cambios que se estn produciendo en tus datos.

Adems, es posible que desees comenzar a explorar los datos antes de haber recopilado todo el conjunto de datos. Si configuras un script que verifique todas estas cosas antes de crear grficos y ejecutar anlisis, es rpido volver a ejecutar el script en los datos completos una vez que los tengas todos. El script acta como un recordatorio de todas las cosas que debes verificar y es mejor para detectar errores tipogrficos que t en una hoja de clculo enorme.
