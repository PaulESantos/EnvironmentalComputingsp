Entonces, ¬øc√≥mo se crea un conjunto de datos ‚Äòfalso‚Äô?

En el caso m√°s sencillo, puedes crear m√∫ltiples vectores y luego combinarlos en un data.frame.

Un atajo √∫til es usar sample, rnorm o runif para crear algunos datos.

sample crea datos ALEATORIOS del tama√±o especificado con o sin reemplazo. Por ejemplo, 10 n√∫meros aleatorios sin reemplazo:

Puedes crear un vector y luego realizar una muestra de √©l:

Tambi√©n puedes realizar muestras n veces, lo cual hace que sea una funci√≥n muy conveniente. Por ejemplo, tomar muestras de los cuatro palos de cartas, 100 veces:

Ten en cuenta que en los ejemplos anteriores, no devuelve data frames, lo cual puede ser o no importante. Utiliza as.data.frame para esto, si es necesario.

Creando datos a partir de una distribuci√≥n conocida

rnorm crea datos a partir de una distribuci√≥n normal.

Por defecto, rnorm extrae de una poblaci√≥n con media = 0 y desviaci√≥n est√°ndar = 1. Podemos cambiar cualquiera de estos valores para obtener una muestra de una distribuci√≥n normal con una media y desviaci√≥n est√°ndar especificadas. Por ejemplo, para obtener 100 n√∫meros de una distribuci√≥n normal con una media de 25 y una desviaci√≥n est√°ndar de 1.5:

runif crea datos a partir de una distribuci√≥n uniforme.

head(data)
## [1] 0.4277050 0.1641431 0.3152610 0.3134036 0.6989719 0.4465205

Al igual que arriba, runif genera valores de una distribuci√≥n con min = 0 y max = 1. Podemos cambiar esto a lo que queramos. Por ejemplo, para obtener 100 n√∫meros aleatorios entre -10 y 5:

Afortunadamente, R tiene pr√°cticamente todas las distribuciones incorporadas para generar muestras. ¬°Esto es realmente √∫til si est√°s teorizando datos antes de comenzar la recopilaci√≥n de datos! Una lista completa se encuentra aqu√≠.

¬øY si tu problema es un poco m√°s complicado?

Por ejemplo, ¬øqu√© sucede si tus datos provienen de cuatro r√©plicas de cada uno de los cinco sitios y deseas recrear un vector para los valores de factor repetidos?

Podr√≠as hacer esto:

Mucho mejor es aprovechar la funci√≥n rep. Esta funci√≥n replica valores en un vector o lista. Se logra el mismo resultado que se mostr√≥ anteriormente con esto.

O, puedes replicar hasta alcanzar una cierta longitud del vector. Para obtener 50 r√©plicas de cada uno de los cuatro sitios, usar√≠amos lo siguiente:

¬°Esto se vuelve cada vez m√°s valioso a medida que aumentas el n√∫mero de repeticiones y/o factores a incluir!

Creando todas las combinaciones de m√∫ltiples factores categ√≥ricos

expand.grid es muy √∫til para crear un marco de datos que contenga todas las combinaciones de todos los niveles de m√∫ltiples factores. Por ejemplo, si hubi√©ramos muestreado cuatro sitios de cada una de las cuatro regiones en cada uno de los tres estados, podr√≠amos usar esto para crear
