Usando bucles

Te encuentras cortando y pegando cdigo de R con frecuencia?

Esto generalmente crear problemas para ti ms adelante. Uno de los principios de una buena codificacin es tratar de reducir al mnimo la repeticin. Hay dos enfoques para organizar tu cdigo y ahorrarte trabajo. El primero es usar funciones y el segundo, cubierto aqu, es usar bucles.

A menudo, queremos realizar tareas repetitivas en las ciencias ambientales. Por ejemplo, es posible que deseemos recorrer una lista de archivos y hacer lo mismo una y otra vez. Hay muchos paquetes en R con funciones que harn todo el trabajo duro por ti (por ejemplo, echa un vistazo a dplyr, tidyr y reshape2 cubiertos aqu). El enfoque de dplyr funciona bien si tus datos son ordenados y estn en un marco de datos. Si tus datos estn en muchos archivos diferentes, entonces un bucle puede ser una solucin ms rpida.

Sintaxis bsica de los bucles

La sintaxis de los bucles es relativamente simple: los componentes esenciales son for(){} donde la parte for() indica cuntas veces se realizan las operaciones dentro de {}.

Considera el siguiente bucle. La primera vez que recorremos el bucle, el valor de i ser igual a 1 y este valor se mostrar con la funcin print. Luego se repetir con i = 2, hasta i = 10, realizando la tarea que se encuentra dentro de {} cada vez.

for (i in 1:10) {
  print(i)
}
## [1] 1
## [1] 2
## [1] 3
## [1] 4
## [1] 5
## [1] 6
## [1] 7
## [1] 8
## [1] 9
## [1] 10

Podemos cambiar el rango de nmeros (1:10) a cualquier cosa que deseemos, no tienen que ser una secuencia de enteros o incluso nmeros. Tambin puedes cambiar i a cualquier valor que desees.

nums <- c(3.2, 890, 0.0001, 400)

for (bat in nums) {
  print(bat)
}
## [1] 3.2
## [1] 890
## [1] 1e-04
## [1] 400
chars <- c("a", "o", "u", "z")

for (bat in chars) {
  print(bat)
}
## [1] "a"
## [1] "o"
## [1] "u"
## [1] "z"

De mayor inters para nosotros es cambiar lo que est dentro de {} o la operacin que estamos realizando en nuestros datos. Podemos insertar cualquier cosa que deseemos aqu. Aqu hay un bucle que imprimir el cuadrado y la raz cuadrada de los nmeros del 1 al 10.

for (i in 1:10) {
  print(i^2)
  print(sqrt(i))
}

A menudo, querremos mantener los resultados que obtenemos de nuestro bucle. La primera opcin es crear un vector o marco de datos vaco y agregar los resultados a l. Esto tarda ms en ejecutarse, pero no importa realmente con bucles simples, aunque puede aumentar los tiempos de espera para estructuras de bucle ms largas y complicadas.

Aqu hay un cdigo que almacenar el cuadrado de los nmeros del 1 al 10 en un nuevo vector llamado x.

x <- vector() # makes a blank vector

for (i in 1:10) {
  y <- i^2 # performs an operation
  x <- append(x, y) # overwrites 'x' with y appended to it
}

Aqu hay un cdigo que almacenar tanto el cuadrado como la raz cuadrada de los nmeros del 1 al 10 en dos columnas de un nuevo marco de datos llamado x2.

x2 <- data.frame(col1 = vector(), col2 = vector()) # makes a blank data frame with two columns

for (i in 1:10) {
  col1 <- i^2 # performs first operation
  col2 <- sqrt(i) # performs second operation
  x2 <- rbind(x2, cbind(col1, col2)) # overwrites 'x2' values including the new row
}

La segunda opcin es crear un vector o marco de datos vaco de dimensiones conocidas y luego colocar los resultados directamente en l. Por ejemplo, si tuviramos un bucle con 10 elementos, podramos almacenar los resultados de cada operacin en un vector con una longitud 10.

x <- vector(length = 10) # makes a blank vector with a length of 10

for (i in 1:10) {
  y <- i^2
  x[i] <- y # places the output in position i in the vector x
}

Alternativamente, guarda los resultados de mltiples operaciones en un nuevo data frame.

x2 <- data.frame(col1 = vector(length = 10), col2 = vector(length = 10)) # makes a blank data frame with two columns and 10 rows

for (i in 1:10) {
  col1 <- i^2 # performs first operation
  col2 <- sqrt(i) # performs second operation
  x2[i, 1] <- col1 # places the first result into row i, column 1
  x2[i, 2] <- col2 # places the second result into row i, column 2
}

Al analizar la estructura de estos datos, tenemos dos variables de respuesta: actividad (nmero de llamadas de murcilagos registradas en una noche) y forrajeo (nmero de llamadas de alimentacin de murcilagos registradas en una noche). Estos datos se recolectaron durante un total de 173 noches de estudio y en 47 sitios diferentes. Hay ocho posibles variables predictoras en el marco de datos, una de las cuales es un factor (Tratamiento.adelgazado), y siete de las cuales son variables continuas (rea.adelgazada, Tiempo.desde.adelgazamiento, Exclusin.adelgazada, Distancia.agua.murray, Distancia.agua.arroyo, Promedio.T, Promedio.H).

Digamos que estamos explorando nuestros datos y nos gustara saber qu tan bien se correlaciona la actividad de los murcilagos con nuestras covariables continuas. Nos gustara calcular el coeficiente de correlacin de Pearson para la actividad y cada una de las covariables por separado. El coeficiente de correlacin de Pearson, calculado con la funcin cor, vara de -1 (correlacin negativa perfecta) a 1 (correlacin positiva perfecta), siendo 0 ninguna correlacin. Almacenaremos todas nuestras correlaciones en un nuevo data frame llamado Correlations.

Primero, usa select de dplyr para hacer una subconjunto de los datos con la variable de respuesta (actividad) y las 5 variables predictoras.

library(dplyr)

Bats_subset <- select(Bats, Activity, Area.thinned:Distance.creek.water)

A continuacin, crea un data frame vaco con dos columnas (el nombre de la variable y la correlacin) y el nmero de filas necesario para almacenar todas las correlaciones.

rows <- ncol(Bats_subset) - 1 # the number of rows needed in our output dataframe

Correlations <- data.frame(
  variable = character(length = rows),
  correlation = numeric(length = rows),
  stringsAsFactors = F
)

Finalmente, podemos utilizar un bucle para calcular cada una de las correlaciones y almacenar la salida en nuestro nuevo data frame.

for (i in 1:rows) {
  temp1 <- colnames(Bats_subset[i + 1]) # retrieves the name of predictor variable
  temp2 <- cor(Bats_subset[, 1], Bats_subset[, i + 1], method = "pearson")
  # calculates the correlation between activity and predictor variable
  Correlations[i, 1] <- temp1 # places the variable name into row i, column 1
  Correlations[i, 2] <- temp2 # places the correlation into row i, column 2
}

## 5  Distance.creek.water -0.09130258

Ahora podemos ver de un vistazo que la actividad est ms fuertemente correlacionada (negativamente) con el rea adelgazada y que no est correlacionada en absoluto con el tiempo desde el adelgazamiento o la temperatura media. Es posible que luego deseemos investigar ms a fondo algunas de estas relaciones con modelos y pruebas estadsticas apropiadas.
